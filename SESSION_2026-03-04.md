| `Backend/src/controllers/authController.js` | register, login, refresh, logout |
| `Backend/src/routes/auth.js` | Route definitions |

### API endpoints
| Method | Route | Auth | Description |
|--------|-------|------|-------------|
| POST | `/auth/register` | None | Create account, returns tokens |
| POST | `/auth/login` | None | Login, returns tokens |
| POST | `/auth/refresh` | None | Rotate refresh token, return new access token |
| POST | `/auth/logout` | None | Revoke refresh token |
| GET | `/auth/me` | Bearer token | Return decoded token payload |
| GET | `/health` | None | Health check |

### Token strategy
- **Access token:** JWT, 15 min TTL, payload `{ userId, email }`
- **Refresh token:** JWT, 7 day TTL, stored in `refresh_tokens` table
- **Rotation:** on refresh, old token is revoked and a new pair is issued

## 4. Login Frontend Page

### Files created / updated
| File | Change |
|------|--------|
| `Frontend/src/lib/auth.js` | `loginUser`, `registerUser`, `logoutUser`, `saveSession`, `getStoredSession` |
| `Frontend/src/components/LoginPage.jsx` | Login / Sign Up tabbed form |
| `Frontend/src/App.jsx` | Auth gate — shows LoginPage or BabySteps based on session |

### Flow
1. App checks `localStorage` for a stored session on mount
2. If no session → renders `LoginPage`
3. On successful login/register → saves tokens to `localStorage`, renders `BabySteps`
4. `onLogout` prop passed to BabySteps revokes the refresh token and clears storage

## How to Run — Complete Setup Guide

> Do these steps **in order**. Steps marked "first time only" can be skipped on subsequent runs.

---

### Step 1 — Install backend dependencies (first time only)

```bash
cd C:\Projects\GotcherApp\Backend
npm install
```

This installs Express, bcryptjs, jsonwebtoken, pg, dotenv, cors, and nodemon.

---

### Step 2 — Create your `.env` file (first time only)

```bash
cd C:\Projects\GotcherApp\Backend
cp .env.example .env
```

Then open `.env` and set a real value for `JWT_SECRET` — make it a long random string, e.g.:

```
JWT_SECRET=some-long-random-string-change-this-in-production
```

The rest of the defaults match the docker-compose config and can be left as-is for local dev.

---

### Step 3 — Make sure TavernTales is not running

Both projects share ports 5432 and 5050. If TavernTales is running, stop it first:

```bash
cd C:\Projects\TavernTales
./stop-services.sh
```

---

### Step 4 — Start all GotcherApp services

```bash
cd C:\Projects\GotcherApp\Backend
./start-services.sh
```

This will:
1. Launch Docker Desktop if it isn't already open
2. Start the `gotcherapp-postgres` and `gotcherapp-pgadmin` containers
3. Start the Node API server (`npm run dev` on port 3001)
4. Start the Vite frontend (`npm run dev` on port 3000)
5. Tail all logs in the terminal with colour-coded output

Press **Ctrl+C** to stop everything.

---

### Step 5 — Run database migrations (first time only)

Open a **second terminal** while `start-services.sh` is running, then:

```bash
cd C:\Projects\GotcherApp\Backend
./run-migrations.sh
```

This applies the SQL migration files in order:
- `V1__create_users.sql` — creates the `users` table
- `V2__create_refresh_tokens.sql` — creates the `refresh_tokens` table

You should see `All migrations applied.` when it completes.

> **Requires `psql` on your PATH.** If you don't have it, you can run the SQL files manually via pgAdmin (see Step 6).

---

### Step 6 — Verify everything is working

| Service | URL | Notes |
|---------|-----|-------|
| Frontend app | http://localhost:3000 | Should show the Login page |
| API health check | http://localhost:3001/health | Should return `{"status":"ok"}` |
| pgAdmin | http://localhost:5050 | Email: `admin@gotcherapp.local` / Password: `admin` |

**To connect pgAdmin to the database:**
1. Open http://localhost:5050 and log in
2. Right-click "Servers" → Register → Server
3. Name: `GotcherApp Local`
4. Connection tab → Host: `postgres` (the Docker service name), Port: `5432`
5. Username: `gotcherapp_app`, Password: `changeme_local`

### Everyday workflow (after first-time setup)

```bash
# Stop TavernTales if running
cd C:\Projects\TavernTales && ./stop-services.sh

# Start GotcherApp
cd C:\Projects\GotcherApp\Backend && ./start-services.sh
```

That's it — Docker, the API, and the frontend all start together.

### Stopping everything

From either project:

```bash
./stop-services.sh
```

This stops both GotcherApp and TavernTales containers and kills all processes on shared ports.
