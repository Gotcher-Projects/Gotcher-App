# S1.5 — Mobile Visual Polish
**Status:** Complete
**Branch:** capacitor-fixes (or new branch off main after S1 merges)
**Depends on:** S1 complete and deployed

## Goal
Audit every tab and form for mobile visual issues and fix them. The app is functional on Android but several areas need responsive polish before S2 (Camera plugin).

## Known Issues Going In (from S1 debugging)
- Text inputs / textareas overflow to the right on narrow screens
- `overflow-x: hidden` added to `html, body` in `index.css` — prevents scrolling but may clip content
- DashboardTab quick log forms: `grid-cols-3` changed to `grid-cols-2 sm:grid-cols-3` (done)
- DashboardTab card padding: `p-4 sm:p-6` (done)
- Tab triggers: `min-w-[60px] text-xs sm:text-sm` (done)

## Root Cause to Investigate First
Check `Frontend/src/components/ui/input.jsx` — if `<Input>` doesn't have `w-full` or has a hard `min-width`, every form using it will overflow. Fix at the component level before auditing each tab.

Also check `Frontend/src/components/ui/textarea.jsx` for the same issue.

**Result (2026-04-29):** Both already had `w-full`. No component-level fix needed. `overflow-x: hidden` was already on `html, body`.

## Fixes Applied (2026-04-29)
- `CradleHq.jsx` header tagline: `hidden sm:block` — hides "Milestone tracker • ..." on narrow screens
- `dialog.jsx` DialogContent: `max-w-[calc(100%-2rem)] sm:max-w-lg` + removed `sm:` from `rounded-lg` — gives 1rem margin each side on mobile
- `DiaperTab.jsx` history: Type, Color, Consistency columns hidden on mobile (`hidden sm:block`) — was overflowing by ~40px
- `FeedingTab.jsx` history: Time column hidden on mobile (`hidden sm:block`) — frees space for Type + Duration

## Fixes to Apply (as issues are found)

### Input/Textarea components
If `input.jsx` / `textarea.jsx` lacks `w-full max-w-full`, add it at the component level — fixes all forms at once.

### Long text that overflows
Add `truncate` or `break-words` / `overflow-wrap: break-word` where needed (names, labels, descriptions).

### Fixed-width containers
Replace any hard `w-[Npx]` or `min-w-[Npx]` with responsive equivalents or remove them.

### Form grids
Any `grid-cols-3` or `grid-cols-2` inside a card with padding should be `grid-cols-1 sm:grid-cols-N` unless the fields are genuinely small.

### Dialog / modals
Check that dialogs don't exceed viewport width — may need `max-w-[calc(100vw-2rem)]` on mobile.
