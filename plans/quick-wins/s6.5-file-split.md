# Session 6.5 — Split BabySteps.jsx by Tab
**Status:** Pending
**Branch:** feature/file-split
**Depends on:** S6 complete (5-tab structure must be in place first)

---

## Why

BabySteps.jsx is ~2530 lines. Every session that touches any tab requires reading the entire
file to get context, which burns context window and makes edits harder to review. After S6's
restructure the natural seam is the top-level tab — one file per tab, one file for the main
shell.

---

## Target Structure

```
Frontend/src/components/
  BabySteps.jsx               ← main shell only (~400 lines after split)
  ui/
    PillNav.jsx               ← moved from bottom of BabySteps.jsx
  tabs/
    DashboardTab.jsx          ← DashboardTab
    MemoriesTab.jsx           ← JournalTab + FirstTimesTab + FirstTimeCard

```

6 new files, BabySteps.jsx shrinks from ~2530 to ~400 lines.

## Shared Imports

Each tab file will need its own imports. Pattern:

```js
// tabs/HealthTab.jsx
import React, { useState } from "react";
import { Card, CardContent } from "@/components/ui/card";
// ... only what that tab uses
```

BabySteps.jsx imports each tab by name:
```js
import DashboardTab from "./tabs/DashboardTab";
import MemoriesTab from "./tabs/MemoriesTab";   // not used yet — see note below
import TrackTab from "./tabs/TrackTab";
import HealthTab from "./tabs/HealthTab";
import DiscoverTab from "./tabs/DiscoverTab";
import PillNav from "./ui/PillNav";
```

**Note on PillNav in TabsContent:** The PillNav + conditional rendering currently lives
inline in BabySteps.jsx's TabsContent blocks. Two options — pick one and note in the plan:

**Option A (simpler):** Keep PillNav + sub-view state + conditional rendering inside each
tab file. Each tab file receives all sub-component props and manages its own `view` state.
BabySteps.jsx just renders `<MemoriesTab data={data} ... />`. Sub-view state moves out of
BabySteps.jsx.

**Option B (current shape):** Keep sub-view state in BabySteps.jsx, pass `view` + `setView`
props into each tab file. Tab files render PillNav + conditional sub-components.

**Recommended: Option A.** Sub-view state (`memoriesView`, `trackView`, etc.) is purely
presentational — it has nothing to do with server data. Moving it into each tab file reduces
BabySteps.jsx state and is cleaner. Each tab file becomes self-contained.

## Tab File Interface (Option A)

```jsx
// tabs/MemoriesTab.jsx
export default function MemoriesTab({ data, week, onAdd, onEdit, onDelete, onUpdateImage, firsts, babyName, onAddFirst, onUpdateFirst, onDeleteFirst, onUpload, onError }) {
  const [view, setView] = useState('journal');
  return (
    <>
      <PillNav options={...} active={view} onChange={setView} />
      {view === 'journal' && <JournalTab ... />}
      {view === 'firsts'  && <FirstTimesTab ... />}
    </>
  );
}
```

BabySteps.jsx TabsContent becomes clean:
```jsx
<TabsContent value="memories" className="mt-4">
  <MemoriesTab data={data} week={week} onAdd={addJournalEntry} ... />
</TabsContent>
```

## Execution Order

Do one tab file at a time to avoid a hard-to-debug big-bang refactor:

1. Create `PillNav.jsx` — simplest, no dependencies
2. Create `DashboardTab.jsx` — no sub-nav, easiest tab

5. Create `HealthTab.jsx` — four sub-components
6. Create `MemoriesTab.jsx` — JournalTab + FirstTimesTab
7. Strip BabySteps.jsx — remove everything that moved out, update imports, verify it compiles

Test in the browser after each tab file is wired up.
