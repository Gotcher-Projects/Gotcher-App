Backend Technologies

Language

Java 21 LTS release

What it is

The programming language the entire backend is written in. Java 21 is a Long-Term Support release, meaning it receives security patches for years.

Why we use it

Spring Boot (our web framework) is a Java-first ecosystem. Java 21 adds modern features like records, pattern matching, and sealed classes that reduce boilerplate significantly.

Where in the code
Backend/src/main/java/ Backend/build.gradle.kts → toolchain
Note Java records are used throughout — GrowthRecord, FeedingLog, AuthPrincipal, all the request/response DTOs. A record is an immutable data class with automatically generated constructor, getters, equals, and toString.
Framework

Spring Boot 3.4.1

What it is

Spring Boot is the glue that holds the backend together. It starts an embedded web server (Tomcat), wires up all dependencies automatically, and routes incoming HTTP requests to the right handler methods.

Why we use it

Without it, you'd manually configure a servlet container, write XML to wire dependencies, and manage connection pools by hand. Spring Boot replaces all of that with annotations and sensible defaults — convention over configuration.

Key annotations to know
AnnotationMeaning
@RestControllerThis class handles HTTP requests. Methods return JSON automatically.
@RequestMapping("/path")All routes in this class start with this prefix.
@GetMapping / @PostMappingHandles GET / POST requests on this method.
@ServiceBusiness logic class — Spring manages its lifecycle.
@RequestBodyDeserialize incoming JSON into a Java object.
@PathVariableBind a {segment} from the URL.
Where in the code
GotcherAppApplication.java application.properties *Controller.java (all 6) *Service.java (all 6)
Security

Spring Security 6

What it is

The authentication and authorization layer. Every HTTP request passes through Spring Security before it reaches a controller. It decides: "Is this person allowed to do this?"

Why we use it

We need to protect every endpoint so users can only see their own data. Spring Security plugs directly into Spring Boot and gives us a clean hook (filters) for our JWT-based auth scheme.

How the auth flow works
1. User logs in → API issues a JWT access token (15 min) + refresh token (7 days)
2. Frontend stores both in localStorage
3. Every request includes Authorization: Bearer <token>
4. JwtAuthFilter intercepts, verifies signature, loads user into SecurityContext
5. Controllers receive the user via @AuthenticationPrincipal AuthPrincipal
6. On token expiry → frontend calls POST /auth/refresh to rotate both tokens
Where in the code
config/SecurityConfig.java security/JwtAuthFilter.java security/JwtUtil.java security/AuthPrincipal.java
Database

JdbcTemplate Raw SQL layer

What it is

A Spring utility that wraps Java's low-level JDBC API. It handles opening/closing database connections, creating prepared statements, binding parameters safely, and converting checked exceptions into unchecked ones — so you can focus on writing SQL.

Why we use it (instead of an ORM like Hibernate)

This app started life as a Node.js project using raw SQL queries. The philosophy carried over: write exact SQL, see exactly what queries run, and avoid the performance surprises of auto-generated ORM queries. For an app of this size, raw SQL is simpler to reason about.

Common patterns you'll see
// Select a list of rows
jdbc.query("SELECT * FROM growth_records WHERE baby_profile_id = ?",
    (rs, i) -> new GrowthRecord(rs.getLong("id"), rs.getString("recorded_date"), ...),
    profileId);

// Insert a row and get the generated ID
jdbc.queryForObject("INSERT INTO ... RETURNING id", Long.class, args...);

// Update / delete — returns rows affected
jdbc.update("DELETE FROM growth_records WHERE id = ? AND baby_profile_id = ?", id, profileId);
Where in the code
growth/GrowthService.java feeding/FeedingService.java journal/JournalService.java auth/AuthService.java
Testing gotcha JdbcTemplate.queryForList has two overloads. When mocking in tests with 5+ arguments, Java may resolve to the wrong overload. If you see PotentialStubbingProblem, add @MockitoSettings(strictness = Strictness.LENIENT) to the test class.
Security

JJWT 0.12.6 JWT tokens

What it is

A Java library for creating and verifying JSON Web Tokens (JWTs). A JWT is a compact, signed string that encodes a payload (user ID, email, expiry) — the server can verify it without a database lookup because the signature proves it hasn't been tampered with.

Why we use it

JWTs let us build a stateless API. The server doesn't maintain session state for every user — the token carries all necessary information. Only refresh tokens (which issue new access tokens) are stored in the database.

JWT structure
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxIiwiZW1haWwiOiJ0ZXN0QGV4YW1wbGUuY29tIn0.signature

header · payload (userId, email, exp) · HMAC-SHA256 signature
Where in the code
security/JwtUtil.java Backend/.env → JWT_SECRET
Database

Flyway Schema migrations

What it is

A database migration tool that tracks which SQL scripts have been applied to a database, and runs new ones in order when the application starts. Think of it as version control for your database schema.

Why we use it

Without Flyway, every developer would need to manually run SQL scripts to keep their local database in sync. On production, a missed script could mean the app crashes on startup. Flyway automates this — it runs exactly once per environment, in the right order.

Migration files (in order)
FileWhat it creates/changes
V1__create_users.sqlusers table (email, password hash, display name)
V2__create_refresh_tokens.sqlrefresh token storage
V3__create_baby_profiles.sqlbaby_profiles (one per user)
V4__create_milestones.sqlachieved milestone keys
V5__create_journal_entries.sqljournal entries (week, title, story)
V6__add_journal_image_url.sqladds image_url column to journal
V7__baseline_gap.sqlintentional placeholder — V7 was skipped
V8__create_baby_profile_display_fields.sqladds parent name, phone to profile
V9__create_growth_records.sqlgrowth tracking table
V10__create_feeding_logs.sqlfeeding session table
V11__growth_records_imperial_units.sqlrenames metric columns → imperial
Important rule Never edit a migration file that has already been applied to any database. Flyway checksums every applied migration and will refuse to start if one has changed. Always create a new migration file instead.
Where in the code
Backend/db/migration/ application.properties → spring.flyway.*

What it is

A Spring Boot module that exposes operational HTTP endpoints. We use it for the /actuator/health endpoint.

Why we use it

Reverse proxies (Caddy, nginx) and load balancers need a way to check "is this app alive?" before routing traffic to it. The health endpoint answers that question: {"status":"UP"}.

Email

Spring Mail SMTP / transactional email

What it is

Spring's email abstraction layer. It wraps JavaMail and provides a simple JavaMailSender bean for sending emails from the application.

Why we use it

The app will eventually send password reset emails and welcome messages. The SMTP configuration is already wired up but disabled by default — leave SMTP_HOST blank to skip email entirely.

Media

Cloudinary Image storage + CDN

What it is

A cloud media management platform. It stores uploaded images, performs on-the-fly transformations (resize, crop, format conversion), and delivers them through a global CDN.

Why we use it

Journal entries support baby photo uploads. Storing images as binary data in PostgreSQL is impractical at any real scale. Cloudinary provides durable cloud storage, automatic format optimization (JPEG → WebP), and a CDN so images load fast globally.

Where in the code
Backend/.env → CLOUDINARY_* vars journal upload endpoint (multipart → Cloudinary → stores URL)
Build Tool

Gradle with Kotlin DSL

What it is

The build tool for the Java backend. It compiles source files, resolves library dependencies, runs tests, and packages everything into a runnable JAR file.

Why we use it

Gradle is the standard build tool for modern Spring Boot projects. The Kotlin DSL (build.gradle.kts) provides type safety and IDE auto-completion that the older Groovy DSL lacks. The Gradle wrapper (gradlew) means no one needs to install Gradle — the correct version downloads automatically on first run.

Testing

JUnit 5 + Mockito Unit testing

What it is

JUnit 5 is the Java test runner — it finds @Test methods, runs them, and reports results. Mockito is a mocking library that replaces real dependencies (like the database) with controllable fakes called "mocks".

Why we use it

Unit tests verify a single class in isolation — no running database, no web server. Tests run in milliseconds and catch regressions early. Mocks let us simulate "what if the database returns empty?" without needing a real database record.

Anatomy of a test
@ExtendWith(MockitoExtension.class)
class GrowthControllerTest {

    @Mock GrowthService growthService;       // fake — behaviour set by when()
    @InjectMocks GrowthController controller; // real controller, gets mock injected

    @Test
    void addRecord_returns201() {
        when(growthService.addRecord(eq(1L), any())).thenReturn(RECORD);

        var result = controller.addRecord(PRINCIPAL, REQUEST);

        assertEquals(HttpStatus.CREATED, result.getStatusCode());
    }
}
Where in the code
Backend/src/test/java/com/gotcherapp/api/ Reports: Backend/build/reports/tests/test/index.html

UI Framework

React 18

What it is

A JavaScript library for building user interfaces out of reusable components. Each component manages its own state and re-renders automatically when that state changes.

Why we use it

React's component model maps naturally to a tabbed app — Journal, Growth, Feeding, and Milestones are each self-contained sections that own their data fetching and local state. It is the most widely used UI library in the industry.

Build Tool

Vite 6

What it is

The frontend build tool and development server. In development it serves files to the browser with hot module replacement (change a file → browser updates instantly, no full page reload). For production, it bundles everything into optimized static files.

Why we use it

Vite is dramatically faster than older tools like Create React App (which used webpack under the hood). It starts in under a second and handles file updates near-instantly.

Dev server at localhost:3000

npm run build

Production bundle → Frontend/dist/

Path alias @/ is an alias for Frontend/src/. So import { cn } from '@/lib/utils' works anywhere without counting ../../ dots.

Styling

Tailwind CSS v3

What it is

A utility-first CSS framework. Instead of writing .card { padding: 16px; border-radius: 8px; } in a separate stylesheet, you compose styles directly on elements using class names like p-4 rounded-lg.

Why we use it

Utility classes keep styles co-located with markup, eliminate the "what do I name this class?" problem, and enforce a consistent design system (8px spacing scale, fixed color palette). Combined with shadcn/ui, it produces a polished result with minimal custom CSS.

Where in the code
Frontend/tailwind.config.js Frontend/src/index.css → @tailwind directives + CSS vars Every .jsx file → className="..."
Components

shadcn/ui + Radix UI

What it is

Radix UI provides accessible, headless (completely unstyled) component behaviors — modal focus trapping, dropdown keyboard navigation, tab panel ARIA roles — all the hard stuff. shadcn/ui layers Tailwind styles on top and gives us components we can copy into our project and own completely.

Why we use it

Accessibility is hard to get right from scratch. Radix handles all ARIA roles and keyboard interactions. Being "copy-paste" (not an installed package) means we own the component code and can tweak any detail without fighting a black box.

Components in use
button.jsx card.jsx input.jsx label.jsx textarea.jsx dialog.jsx tabs.jsx select.jsx badge.jsx switch.jsx checkbox.jsx separator.jsx
The cn() helper import { cn } from '@/lib/utils' — merges Tailwind class names safely, resolving conflicts (e.g., two padding classes → last one wins). Use it any time you conditionally combine class strings.
Where in the code
Frontend/src/components/ui/ Frontend/src/lib/utils.js
Icons

lucide-react

What it is

A React icon library with 1,000+ SVG icons available as components. Each icon is a self-contained React component that renders a crisp SVG at any size.

Why we use it

Icons communicate meaning faster than text alone. lucide-react is the icon set used by shadcn/ui, so it's already in the dependency tree and the style is consistent with the rest of the UI.

Charts

recharts

What it is

A composable charting library for React, built on D3 under the hood. Charts are expressed as JSX components — you compose them like any other React component.

Why we use it

The Growth Tracker needs a line chart to visualize baby weight, height, and head circumference over time. recharts integrates naturally into the React component model without requiring imperative D3 code.

How data is shaped for the chart
// Each growth record becomes one data point:
{ date: "11/03", weight: 7.5, height: 20.0, head: 13.5 }

// Then passed to the chart:
<LineChart data={chartData}>
  <Line dataKey="weight" stroke="#f472b6" name="Weight (lbs)" />
  <Line dataKey="height" stroke="#60a5fa" name="Height (in)" />
  <Line dataKey="head"   stroke="#4ade80" name="Head (in)" />
</LineChart>

Export

jsPDF Client-side PDF generation

What it is

A JavaScript library that generates PDF files entirely inside the browser — no server needed. It draws text, images, and shapes onto a virtual PDF canvas, then triggers a file download.

Why we use it

The Memory Book feature lets parents export their baby's journal as a PDF keepsake. Generating it client-side means no sensitive journal content is sent to a server for rendering, and there's no added server load.

Where in the code
Frontend/src/lib/pdf.js
Paywall seam pdf.js exports two functions: generatePdf() builds the PDF object, and downloadPdf() triggers the save. The split is intentional — a future paywall check sits between the two.
Browser API

localStorage

What it is

A browser built-in API that stores key-value string data persistently — it survives page refreshes and browser restarts, but is private to the current browser (not synced across devices).

Where we use it (two places)
KeyWhat it storesWhy
gotcher_session JWT access token + refresh token Keeps the user logged in across page refreshes without a server-side session
gotcher_active_feed Feed start time + type If the page is refreshed mid-feed, the timer resumes from the saved start time instead of resetting to zero
Where in the code
Frontend/src/lib/auth.js → saveSession / getStoredSession BabySteps.jsx → Feeding tab timer logic
Security note localStorage is readable by any JavaScript on the page (XSS vulnerability). For a future hardening pass, consider moving auth tokens to httpOnly cookies, which are inaccessible to JavaScript.

Docker Compose Local dev environment

What it is

Docker runs applications in isolated containers. Docker Compose lets you define multiple containers (PostgreSQL + pgAdmin) in a single YAML file and start them all with one command.

Why we use it

Docker eliminates the "works on my machine" problem. Every developer gets an identical PostgreSQL 16 setup regardless of their OS — no manual installation, no version conflicts, no leftover data from other projects.
